# Troubleshooting

## 1) Ultralytics path issues / data.yaml

### Симптомы
- `data.yaml not found`
- `val/images not found`
- обучение стартует, но не видит датасет

### Проверка
1. Убедитесь, что `data.yaml` существует.
2. Проверьте относительные пути `train:`/`val:` внутри `data.yaml`.
3. Для Windows используйте корректные слэши и кодировку путей.
4. Проверьте, что `images/*` и `labels/*` соответствуют друг другу по именам.

### Частая причина
Ultralytics может опираться на собственные settings/path-правила. Если есть mismatch между фактическим путём датасета и тем, что задано в конфиге/окружении, используйте абсолютный путь к `data.yaml`.

## 2) CUDA / OOM

- Уменьшите `batch`.
- Уменьшите `imgsz`.
- Попробуйте `device=cpu`.
- Проверьте совместимость torch/cuda/driver.

## 3) Ошибки аннотаций

- bbox должен быть в диапазоне [0,1].
- class_id должен попадать в диапазон `names[]`.
- пустые/битые label/image надо исправить до обучения.

## 4) Падение вкладок UI

- Откройте вкладку «Задачи» и логи.
- Используйте error widget traceback + папку логов.
