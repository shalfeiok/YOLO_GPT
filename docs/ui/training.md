# Вкладка «Обучение»

## Что это
Основной экран запуска обучения Ultralytics YOLO с параметрами, прогрессом, консолью и метриками.

## Где в UI
Sidebar → **Training**.

## Основные параметры

| Параметр | Смысл | Рекомендовано | Риски |
|---|---|---|---|
| model_name | Базовая модель (например yolo11n.pt) | Начинать с n/s версии | Слишком тяжёлая модель на слабом GPU |
| weights_path | Дообучение от существующих весов | Использовать best.pt от похожей задачи | Несовместимые классы |
| epochs | Число эпох | 50–150 для первого прохода | Недо/переобучение |
| batch | Размер батча | максимально возможный без OOM | CUDA OOM |
| imgsz | Размер изображения | 640 как база | Мелкие объекты могут требовать больше |
| patience | early stopping | 20–40 | Слишком маленькое значение остановит рано |
| workers | dataloader workers | 0..4 на Windows | При cache=True принудительно 0 |
| optimizer | оптимизатор | `auto` по умолчанию | Неподходящий оптимизатор |

## Workflow
1. Выберите модель/веса.
2. Укажите путь к датасету (или нескольким).
3. Настройте параметры.
4. Запустите обучение.
5. Следите за прогрессом, консолью, метриками.
6. Откройте артефакты в runs/train/*.

## Ошибки
- CUDA incompatibility: сервис автоматически делает fallback на CPU в части сценариев.
- Ошибки разметки: проверьте dataset inspector и labels.
