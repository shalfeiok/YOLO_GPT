# Отчёт: унификация настроек и применение рекомендаций Training Advisor

## Какие проблемы были

1. Вкладка «Советник по обучению» должна быть явно доступна в общей навигации и стеке вкладок.
2. В Training UI были остатки локального состояния (`_advanced_options`) и риск рассинхронизации между UI-полями и тем, что реально уходит в запуск.
3. После применения рекомендаций советника не было явного принудительного обновления Training UI из `settings_store` и удобного подтверждения diff после фактического применения/undo.
4. Не хватало явного лога параметров перед `model.train(**train_kw)`, чтобы быстро проверить, что в запуск ушли именно обновлённые `imgsz/mixup/mosaic/close_mosaic/patience`.

## Какие изменения внесены

1. Навигация и доступность вкладки:
   - обновлена кнопка в сайдбаре: короткий RU label «Советник» и русскоязычный tooltip.
2. Единый источник правды для запуска обучения:
   - добавлен модуль `app/ui/views/training/train_args.py` с явной функцией `build_training_launch_args(settings_snapshot, ...)`;
   - запуск обучения в `TrainingView._start_training()` теперь собирает аргументы только из snapshot `settings_store` (через `SettingsController.training()`), а затем передаёт их в `ViewModel` единым объектом;
   - удалён локальный `_advanced_options` как источник состояния.
3. Применение рекомендаций/undo:
   - после `ApplyAdvisorRecommendationsUseCase.execute()` TrainingView сразу перечитывает store и перезаливает значения в UI (`_apply_training_settings_to_ui()`);
   - показывается diff-preview именно фактически применённых изменений;
   - для undo добавлен аналогичный UI-refresh из store + diff-preview отката.
4. Надёжность/прозрачность запуска в `TrainingService`:
   - whitelist advanced-опций вынесен в явную константу `ALLOWED_ADVANCED_TRAINING_KEYS`;
   - добавлен лог перед `model.train(**train_kw)` c ключевыми параметрами:
     `imgsz, batch, epochs, patience, workers, mosaic, mixup, close_mosaic`.

## Как проверить вручную

1. Запустить приложение и открыть вкладку **«Советник»** в левом меню.
2. Выполнить анализ (или загрузить рекомендации), затем перейти на вкладку **«Обучение»**.
3. Нажать **«Применить рекомендации советника»**.
4. Убедиться, что поля в UI обновились (например `imgsz`, `patience`, расширенные `mosaic/mixup/close_mosaic`).
5. Нажать **«Старт обучения»**.
6. Проверить в логах/консоли строку старта train-аргументов — там должны быть применённые значения.
7. Нажать **«Отменить применение»** и убедиться, что UI возвращается к предыдущему snapshot.

## Что должно быть видно в консоли

Пример диагностической строки перед запуском `model.train(**train_kw)`:

`Starting YOLO training with args: imgsz=960 batch=8 epochs=120 patience=40 workers=2 mosaic=0.3 mixup=0.15 close_mosaic=15`

И в выводе Ultralytics `engine\trainer: ...` должны фигурировать эти же значения.
